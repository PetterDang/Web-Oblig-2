Git / GitHub

GitHub Inc er et web-basert service som tar i bruk Git som er et system for distribuert versjonskontroll. I GitHub er alle distribuerte versjonskontroll og SCM funksjoner inkludert. I tilleg har GitHub flere funksjoner. Ved å ha distribuert versjonskontroll gjør det det mulig å se alle endringer gjort på filene. Dette gjør det lettere å samarbeide med andre utvikliere.

Når du laster opp en fil "Committer" du en ny versjon av en fil. Hver gang du laster opp den samme filen med endringer blir filen oppdatert og du kan se endringene gjort over tid. Ved bruk av versjonskontrol blir det mulig å hente tidligere versjoner av filen som har blitt lastet opp. Det som gjør GitHub så lett for andre utvikliere er at det er mulig for andre brukere å laste opp en ny versjon av filen. Dette heter en "Commit Request", ved bruk av dette kan flere utvikliere skrive kode sammen og oppdatere de. Endringer må bli godtatt av eieren av filen.

Github kan ha "Open Source" kode eller "Closed Source" kode. Ved å ha "Open Source", som er mest vanlig ved bruk av GitHub er det mulig for alle å se koden din. Hvis den er "Open Source", så kan alle kopiere koden din og, de kan da bruke den i sine egne prosjekter ved å videreutvikle den, eller se på den og lære.
